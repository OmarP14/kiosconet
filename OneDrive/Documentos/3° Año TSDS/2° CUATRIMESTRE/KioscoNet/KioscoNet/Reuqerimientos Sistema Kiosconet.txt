Análisis: Requerimientos – KioscoNet
NºRequerimientoTipoDescripción (Relevamiento)RF1Registrar ventas con cálculo automático del vuelto.FuncionalEl cliente indicó que quiere que el sistema calcule el vuelto automáticamente en cada venta.RF2Actualizar inventario en cada transacción.FuncionalEl dueño mencionó que quiere que cada venta descuente stock sin hacerlo manualmente.RF3Generar comprobante impreso o digital (ticket/PDF).FuncionalEl cliente pidió poder entregar comprobantes en papel o en digital.RF4Permitir múltiples listas de precios (minorista/mayorista).FuncionalEl kiosco vende a distintos tipos de clientes, por lo que requiere listas de precios diferenciadas.RF5Registrar ingresos y egresos de dinero.FuncionalNecesidad de registrar movimientos de caja además de las ventas.RF6Mostrar el saldo disponible en cualquier momento.FuncionalEl cliente quiere ver cuánto dinero hay en caja en tiempo real.RF7Guardar historial de movimientos con fecha, hora y usuario.FuncionalPermite control y auditoría de la caja.RF8Registrar mercadería ingresada al sistema, incluyendo proveedor, cantidad, precio de compra y fecha de ingreso, permitiendo asignar el producto a una o varias listas de precios (minorista, mayorista u otras).FuncionalEl cliente necesita registrar los datos completos de la mercadería al ingresar (proveedor, precio de compra, cantidad) y que el sistema permita definir a qué lista de precios se asignará (minorista, mayorista, promociones).RF9Descontar productos automáticamente al registrarse una venta.FuncionalEvita errores de control de inventario.RF10Emitir alertas de stock bajo.FuncionalNecesita avisos cuando los productos lleguen a stock mínimo.RF11Registrar fechas de vencimiento y generar alertas.FuncionalEl cliente quiere evitar pérdidas por vencimiento de productos.RF12Crear usuarios con distintos roles (Administrador/Vendedor).FuncionalEl cliente desea que cada empleado tenga permisos distintos.RF13Cada usuario debe tener credenciales únicas.FuncionalNecesidad de seguridad en accesos.RF14Registrar acciones de cada usuario.FuncionalEl dueño quiere saber qué acciones realiza cada empleado.RF15Guardar datos de clientes y proveedores.FuncionalNecesidad de llevar agenda de clientes y proveedores.RF16Administrar cuentas corrientes.FuncionalEl cliente mencionó que muchos clientes compran fiado.RF17Reporte de ventas diarias, semanales y mensuales.FuncionalRequiere informes de ventas para control de ingresos.RF18Mostrar productos más vendidos.FuncionalAyuda a conocer tendencias de consumo.RF19Reportes de stock valorizado y rentabilidad.FuncionalFacilita conocer ganancias y valor del inventario.RF20Integrar medios de pago electrónicos.FuncionalEl cliente usa tarjetas y billeteras digitales.RF21Ser compatible con impresoras de tickets.FuncionalNecesidad de imprimir comprobantes de venta.RF22Calcular automáticamente comisiones (si aplica).FuncionalEn caso de pagos electrónicos, calcular comisiones.RNF1Interfaz amigable y de fácil aprendizaje.No FuncionalEl cliente requiere que el sistema sea fácil de usar para cualquier empleado.RNF2Tiempo de respuesta < 2 segundos por operación.No FuncionalNecesidad de rapidez en atención al cliente.RNF3Seguridad en acceso y datos.No FuncionalImportante para proteger la información del negocio.RNF4Base de datos respaldable y recuperable.No FuncionalPermite no perder datos por fallos del sistema.RNF5Adaptabilidad a regulaciones fiscales.No FuncionalEl sistema debe poder adaptarse a cambios de normativas.
